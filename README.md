# Bitfonter
- [WEB версия](https://alexgyver.github.io/Bitfonter/)
- [Релиз](https://github.com/alexgyver/Bitfonter/releases/latest)

Конвертер шрифтов в растровый формат Bitfont, с которым умеет работать например GyverGFX2 и библиотеки на её основе:

- Алфавит с любыми символами до 0xFFFF (16 бит)
- 13 встроенных web шрифтов
- Поддержка внешних шрифтов из файлов
- Bold и Itailc модификация
- Monospace модификация (по ширине самого широкого символа)
- Редактор растра - пиксели на превью кликабельные
- Вывод в файл или массив
- 3 варианта кодирования:
  - RLE упаковка
  - RLE упаковка со сглаживанием (полупрозрачные пиксели)
  - Битмап столбиками 8 пикс LSB вверх, сверху вниз слева направо

## Ограничения
- Длина алфавита: 255 символов
- Количество диапазонов алфавита: 255
- Размер массива: 16-бит индекс (до 65535)

## Формат Bitfont
### Хэдер
Общая информация

```
0 font type (1 rle bitpack, 2 rle bitpack smooth, 3 bitmap v8-col)
1 space width
2 line height
3 glyph amount
4 ranges amount
```

### Диапазоны
Массив диапазонов кодов символов

```
from        2 bytes LE
length      1 byte
```

### Индексы
Массив из `glyph amount + 1` индексов по 2 байта LE. Индексы считаются от конца массива индексов, то есть от начала данных первого глифа. Для глифа с номером `i`:

```
data   = glyphData + index[i]
length = index[i + 1] - index[i]
```

### Глиф
Данные глифа символа

```
0 width
1 height
2 interval - отступ после символа
3 offset - вертикальный отступ начала координат
4+ glyph data
```

## Формат данных глифов
Во всех режимах пиксели обходятся столбцами сверху вниз слева направо.

### RLE
Поток разбивается на серии одинаковых по цвету пикселей длиной от 1 до 32. Каждая серия занимает 6 бит:

```
vlllll
```

- `v` - значение пикселя: `0` пустой, `1` закрашенный
- `lllll` - длина серии минус 1, от `0` до `31`

6-битные чанки записываются подряд от старших битов к младшим, по 4 чанка в 3 байта. Последняя тройка байтов может быть неполной - её размер задаётся длиной данных глифа из таблицы индексов или высчитывается как остаток при распаковке.

### RLE smooth
Такое же дробление на 6-бит чанки, но добавляется полупрозрачный бит:

```
vsllll
```

- `v` - значение основного пикселя: `0` пустой, `1` закрашенный
- `s` - флаг сглаженного начала серии
- `llll` - длина серии минус 1, от `0` до `15`

При `s = 1` первый пиксель серии имеет полупрозрачность, а оставшиеся `length - 1` пикселей имеют значение `v`.

### Bitmap
Растр разбивается на вертикальные группы по 8 пикселей, LSB сверху. Размер данных равен `width * ceil(height / 8)` байт.
